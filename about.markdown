#### Tools of choice

I use [Aquamacs](http://aquamacs.org/) on a [Macbook](http://www.apple.com/macbook) running [Mac OS X 10.6](http://www.apple.com/macosx/leopard/) to work at home, [Emacs](http://www.gnu.org/software/emacs/) on an [eeePC 1005ha](http://????) running [Ubuntu 10.04](http://www.ubuntu.com/) when I need to work outside and [Ubuntu Server](http://www.ubuntu.com/) on servers. I use [Chrome](http://www.google.com/chrome) for browsing and debugging, and [ZSH](http://www.zsh.org/) is my default shell these days.

My web framework of choice for small applications is [Sinatra](http://www.sinatrarb.com/), and most of the time I use [Python](http://www.python.org/) or [Ruby](http://www.ruby-lang.org/) as general-purpose programming languages. I don't really like [PHP](http://php.org), but it's useful sometimes. I'm studying [Erlang](http://www.erlang.org/) now for writing server-like concurrent-based applications, and I'm learning more of [Javascript](http://en.wikipedia.org/wiki/JavaScript) by playing with [CouchDB](http://couchdb.apache.org/).
